# CLAUDE.md — 恋詠ノ書（恋愛タロット鑑定アトリエ）運用ルール

> このファイルは **Claude Code 向けの指示書**です。
> このZip（またはフォルダ）を受け取ったら、まずこのファイルと `HANDOFF.md` を読み、
> このツールの持ち主（以下「オーナー」）の作業を、日本語で1ステップずつ手伝ってください。
> オーナーは非エンジニアの想定です。

## あなた（Claude Code）の役割
このツール「恋詠ノ書」を、オーナー自身のGitHubに公開できるよう手伝うこと。

## このファイルの適用範囲（オーナーの環境を壊さないために）
- この `CLAUDE.md` は **このツールのフォルダ内だけ** に適用される指示です。
- オーナーが既に持っている**他の `CLAUDE.md`**（例：`~/.claude/CLAUDE.md` や別プロジェクトのもの）を、
  **変更・移動・上書き・削除してはいけません。** 既存設定は必ずそのまま残すこと。
- このZipは、必ず**新しい空のフォルダ**に展開してください。展開先に既存の `CLAUDE.md` などがある場合は、
  **上書きせず、いったん止めてオーナーに確認**してください。
- `/init` 等で `CLAUDE.md` を新規生成し直さないこと。このフォルダの `CLAUDE.md` が正です。

## 大事な前提：このツールは APIキー不要
- 恋詠ノ書は **AIのAPIを直接呼びません**。利用者が自分のChatGPT / Claudeに貼って使う仕組みです。
- したがって **Cloudflareもキー設定も不要**。`index.html` 1ファイルの完全自己完結。
- キーに関する設定は一切ありません。もしオーナーがAPIキーの話をしても、このツールには不要と伝えてください。

## 公開までの手順（詳細は HANDOFF.md）
1. このZipを展開し、その中身をオーナーのGitHubに**公開リポジトリ**として作成する
   （リポジトリ名の例 `koi-tarot-atelier`）。
2. GitHub Pages（mainブランチ / ルート）を有効化し、公開URLをオーナーに伝える。
3. これで完成。設定作業は他にありません。

## デプロイ後にオーナーへ提案すること（精度チェック・改善）
公開できたら、次を「やってみますか？」と**オーナーに提案**してください。
勝手に大きく変えず、**提案 → 合意 → 実施 → プレビューで確認 → push** の順で進めること。

### A. 出力の精度チェック
- このツールは「AIに渡すプロンプト」を作ります。実際にそのプロンプトを ChatGPT / Claude に貼って、
  出てきた鑑定書をオーナーと一緒に確認する。
- 観点：相談者名で書けているか／断定を避けているか／見出し【〇〇について】が付いているか／
  冒頭・締めの固定文が入っているか（＝ツール内蔵の「納品前チェック」でも自動確認できる）。
- 直したい時は `index.html` 内の `SYSTEM_PROMPT`（占い師AIへの指示文の本体）を調整する。

### B. デザイン・機能の修正
- 色・タイトル：`index.html` 冒頭 `<style>` の `:root`（`--gold` などの変数）。
- 入力項目・納品前チェックの条件：`index.html` のフォーム／`checkBtn` 周り。
- 変更は**ローカルでプレビュー確認 → 問題なければ push** の順。結果をオーナーに見せてから確定する。

すべて `index.html` の中にあります。迷ったら `HANDOFF.md` を参照してください。
