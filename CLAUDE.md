# CLAUDE.md — 恋詠ノ書（恋愛タロット鑑定アトリエ）運用ルール

> このファイルは **Claude Code 向けの指示書**です。
> このZip（またはフォルダ）を受け取ったら、まずこのファイルと `HANDOFF.md` を読み、
> このツールの持ち主（以下「オーナー」）の作業を、日本語で1ステップずつ手伝ってください。
> オーナーは非エンジニアの想定です。

## あなた（Claude Code）の役割
このツール「恋詠ノ書」を、オーナー自身のGitHubに公開できるよう手伝うこと。

## 大事な前提：このツールは APIキー不要
- 恋詠ノ書は **AIのAPIを直接呼びません**。利用者が自分のChatGPT / Claudeに貼って使う仕組みです。
- したがって **Cloudflareもキー設定も不要**。`index.html` 1ファイルの完全自己完結。
- キーに関する設定は一切ありません。もしオーナーがAPIキーの話をしても、このツールには不要と伝えてください。

## 公開までの手順（詳細は HANDOFF.md）
1. このZipを展開し、その中身をオーナーのGitHubに**公開リポジトリ**として作成する
   （リポジトリ名の例 `koi-tarot-atelier`）。
2. GitHub Pages（mainブランチ / ルート）を有効化し、公開URLをオーナーに伝える。
3. これで完成。設定作業は他にありません。

## カスタマイズ（依頼されたら）
- 鑑定のルール・文体 → `index.html` 内の `SYSTEM_PROMPT`
- 入力項目・納品前チェック → `index.html` のフォーム／`checkBtn` 周り
- 色・タイトル → `index.html` 冒頭 `<style>` の `:root`

すべて `index.html` の中にあります。迷ったら `HANDOFF.md` を参照してください。
